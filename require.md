# 需求矩阵（最终版）

| 序号 | 模块      | 核心逻辑    | 规则说明与可视化                                                                               |
|:---|:--------|:--------|:---------------------------------------------------------------------------------------|
| 1  | 任务格式规范  | 统一格式管控  | 统一格式：序号. 任务描述@日期@状态。示例：1. 修复登录Bug@4.15；2. 梳理架构@4.16@未完成                                |
| 2  | 格式解析规则  | 字段强容错解析 | 1. 日期可选：未写@日期，默认=消息发送当天；2. 状态可选：未写@状态，默认=已完成；3. 仅任务描述：自动补全@发送当天@已完成；4. 支持缺字段、缺@、多空格弱解析 |
| 3  | 对话内容识别  | 聊天/任务区分 | 智能区分消息类型：todolist 格式→进入任务解析；常规聊天→不触发任务流程，仅正常对话响应                                       |
| 4  | 环境门控    | 场景权限管控  | 日历配置：配置化日历统一走 config.yaml                                                              |
| 5  | 环境门控    | 场景权限管控  | 节假日阻断：按年度节假日 JSON 文件阻断非工作日                                                             |
| 6  | 环境门控    | 场景权限管控  | 钉钉考勤状态：若接口返回当前用户处于“请假”状态，自动静默退出                                                        |
| 7  | AI大脑    | 专业级润色   | 按「今日工作」「明日计划」格式化，补全术语，短句扩写为正式职场汇报                                                      |
| 8  | 执行枢纽    | 双模确认机制  | 1. 触发后进入“待命状态”，Agent 弹出预览并开启15分钟倒计时；2. 手动模式：输入Y/点击确认立即发送；3. 自动模式：15分钟无响应且非请假，自动执行      |
| 9  | 可视化增强   | 数据可视化卡片 | Markdown 嵌入任务分布表与状态                                                 |
| 10 | 动态感知    | 对话实时感知  | 实时感知用户分阶段对话输入的 todolist，增量合并、实时刷新任务列表，无文件依赖，机器人是通过Stream模式，websocket来连接的，没有回调          |
| 11 | MCP隔离调用 | Agent定位 | Agent 作为“大脑”不碰底层 API                                                                   |
| 12 | MCP隔离调用 | 解耦调用    | 生成内容后通过本地进程唤起 mcp_server 发日志，极致解耦                                                      |
| 13 | 数据源适配   | 钉钉机器人交互 | 全程通过钉钉机器人对话获取用户任务                                                        |
| 14 | 多用户管理   | 用户标识与存储 | 提取并持久化存储 user_id，作为用户唯一标识，关联任务、模板、配置                                                   |
| 15 | 多用户模板配置 | 个性化模板适配 | 按 user_id 绑定不同 template_id，支持日志发送 sort、key 字段差异化配置                                     |
